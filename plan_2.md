## 需求一：保存的标注 json 路径可自由选择，且不再限制 50 条

### 现状

* 目前保存标注 json 的路径是固定的，或不可由用户自由选择。
* 当前标注数量可能存在“最多 50 条”的限制。

### 目标

1. 保存标注结果时，用户可以自行选择 json 的保存路径，而不是写死固定路径。(使用cookie之类的方式鉴别用户保存上次状态)
2. 去掉“最多 50 条”的限制，允许保存超过 50 条标注。
3. 如果原先前端、后端、状态管理、数据校验中都存在 50 条限制，需要全部移除，避免只改一处导致仍然报错。
4. 若用户未选择路径，给出清晰提示；若项目已有默认路径机制，可保留默认路径，但必须允许用户改写。

### 实现要求

* 检查并修改：

  * 前端 UI 的路径输入/选择逻辑
  * 保存函数
  * 后端接口或本地写文件逻辑
  * schema / validator / form rules / 常量中的 50 条限制
* 若是桌面端应用，优先使用系统文件选择器。
* 若是 Web 项目且浏览器环境不允许直接写本地任意路径，请基于现有架构实现“用户指定导出文件名/位置”的合理方案，并说明受限点。
* 保存时保证 json 编码正确，避免中文乱码，建议 utf-8。
* 保存失败时给出可读错误信息。

### 验收标准

* 可以指定任意合法 json 保存位置
* 可以保存 50 条以上标注，例如 80 条、120 条
* 不会因数量超过 50 条而报错、截断或 UI 异常

### 目标

在现有标注内容中新增一个字段：`mental_reasoning`。

### 字段定义

“根据视频、音频、Current State，调用 VLM 推理得出行为主体如何理解世界并据此行动。它是一种可由多模态线索推断、能约束动作选择的潜在状态表征。”

请将它作为正式标注字段加入 UI、数据结构和保存结果中。

### 具体要求

1. 在标注界面中新增 `mental_reasoning` 的输入或展示区域，风格与现有标注字段保持一致。
2. 在保存的 json 中加入该字段。
3. 若项目中存在：

   * 标注对象类型定义
   * 表单 schema
   * 默认值初始化
   * 编辑/回显逻辑
   * 导入导出逻辑
     这些地方都要同步补齐 `mental_reasoning`。
4. 如果当前系统支持自动生成或调用模型推理，请为 `mental_reasoning` 预留接入位置，接口命名保持清晰。
5. 如果当前还没有实际的 VLM 调用链路，先把字段和接口占位设计好，不要伪造后端能力。

### 建议字段说明（可用于 tooltip / placeholder / help text）

可参考以下文案：

* “结合视频、音频和 Current State，描述行为主体如何理解当前环境、关注什么、预期什么，并据此做出动作选择。”
* “这不是表面动作复述，而是对主体潜在认知状态、判断依据和行动约束的概括。”

### json 示例

请确保保存结果中支持类似结构：

```json

  "start_time": 3.2,
  "end_time": 8.6,
  "action": "pick up cup",
  "current_state": "cup is on the table, person is looking at it",
  "mental_reasoning": "The subject likely infers that the cup is the intended object of interaction, attends to its location and graspability, and chooses an action consistent with picking it up."
}
```

### 验收标准

* UI 中可以新增、编辑、查看 `mental_reasoning`
* 保存后的 json 中存在 `mental_reasoning`
* 重新加载已有标注时，`mental_reasoning` 能正确回显
* 不会因为旧数据没有该字段而崩溃，应做好兼容（例如默认空字符串）

## 需求三：视频标注合法长度改为 2s 到 15s

### 现状

当前视频标注片段的合法长度范围不是我要的，需要调整。

### 目标

将单条视频标注片段的合法长度修改为：

* 最短 2 秒
* 最长 15 秒

### 实现要求

1. 修改所有相关校验逻辑，而不只是 UI 文案：

   * 前端表单校验
   * 时间选择器限制
   * 后端校验
   * schema / constants / helper function
2. 合法判断应基于：
   `duration = end_time - start_time`
3. 新规则：

   * `duration >= 2`
   * `duration <= 15`
4. 对非法片段给出明确提示，例如：

   * “标注片段长度不能小于 2 秒”
   * “标注片段长度不能大于 15 秒”
5. 若存在边界精度问题（浮点数误差），请做合理处理，避免 1.999999 或 15.0000001 造成误判。

### 验收标准

* 2 秒片段可通过
* 15 秒片段可通过
* 小于 2 秒被拦截
* 大于 15 秒被拦截
* 前后端规则一致，不会出现前端能提交、后端又报错的情况
